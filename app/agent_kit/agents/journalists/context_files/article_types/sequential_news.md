A sequential news article converts pre-vetted factual anchors into a clear, chronological narrative. The goal is to reduce reasoning overhead by following the source order directly instead of reorganizing into an inverted pyramid.

WORKFLOW (FOLLOW IN ORDER):
1. Read anchors in the exact order they are provided in ANCHOR CONTEXT.
2. Build the lead from the first 1-2 anchors in that sequence.
3. Continue paragraph-by-paragraph in the same sequence.
4. Write one sentence (or one tight pair) per factual anchor; do not merge unrelated anchors into one sentence.
5. For each sentence, copy the `source_url` from **that same anchor line** only — never attach one anchor's link to another anchor's facts.
6. Include inline timestamp links where source_url is provided so the reader lands where that topic **starts** in the video.
7. Close with concrete outcomes, next steps, or pending decisions.

RULES:
- Do not invent facts, quotes, chronology, or context not present in the anchors.
- Do not reorder events unless two adjacent anchors are clearly the same topic and can be merged.
- Never reuse an early-meeting timestamp link on a later topic; each link must match the anchor it cites.
- Keep transitions simple and factual ("later in the meeting", "at 01:12:30", etc.).
- Prefer clarity over flourish; concise, direct reporting is required.
- If a section has no substantive decisions, summarize briefly and move on.

PURPOSE:
Produce a publication-ready local news article that is easy to verify against the meeting record because narrative flow matches anchor chronology.
